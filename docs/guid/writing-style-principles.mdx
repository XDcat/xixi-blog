# 我的文档写作风格原则

## 一、核心原则

我的文档写作遵循《金字塔法则》：**先结论，后展开；先重点，后细节**。目标不是把内容堆满，而是让读者先抓住主线，再顺着结构理解细节。

> <mark>总分清晰，重点前置，标题即内容。</mark>

## 二、标题与结构

### 2.1 标题要有信息量

标题不是编号，而是段落摘要。好的标题应该让人一眼知道这一节在讲什么。

### 2.2 标题结构要体现主次关系

标题结构本身就是逻辑结构。常见的好结构，不是“把内容分得很细”，而是“把关系说得很清楚”。

```mdx
# 大标题
# 一、一级标题
## 1.1 二级标题
### 1.1.1 三级标题
#### a. 四级标题
```

> **注意：** 通常只使用三级标题。万不得已的时候可以使用四级标题，不要有五级或者更高的标题。这样做是为了让段落清晰，不要过多也不要过少。

> **注意：** 标题的划分是为了有更好的行文思路，而不仅仅是为了区分不同内容。这意味着每个标题下需要足够多的内容。错误的案例：将三句话分为了三个子标题，而是应该放在一个段落中，用不同的标号或者表格区分。

### 2.3 标题层级要控制

- 不要为了分层而分层；同一逻辑下的内容应放在同一段落或同一节里
- 每个标题下面要有足够内容，避免过度切碎
- 如果几个句子表达的是同一个论点，就应当合并，而不是拆成多个标题

### 2.4 标题要服务结构

- 先讲主线，再补充细节
- 结论优先出现，解释随后展开
- 同一层级的内容尽量保持同等粒度

## 三、表达风格

- 风格要生动、有趣，但保持客观专业
- 可以适当使用比喻、对比、转折，但不要喧宾夺主
- 通过结构、例子、对比来讲清楚
- 避免过度修辞和空泛表达

**表达示例：**

- 不好：这件事很重要。
- 更好：这件事很重要，因为它决定了读者是否能在第一时间抓住主线。

## 四、MDX 使用偏好

- 优先使用 MDX
- 适度使用 **粗体**、<span style={{ color: 'red' }}>字体着色</span>、<mark>段落高亮</mark>
- 适合用表格时优先用表格
- 提示块可用于强调结论或注意事项
- 强调手段要服务于信息表达，不要为了“好看”而使用

**常用案例：**

### 4.1 粗体

用于强调核心概念或关键结论，例如：**先结论，后展开**。

### 4.2 字体着色

用于提醒、警示或特别重要的内容，例如：

<span style={{ color: 'red' }}>不要为了分层而分层。</span>

### 4.3 段落高亮

这里的“段落高亮”指的是**提示块**，不是 `mark`。用于强调整段结论、原则或提醒，例如：

:::tip[写作原则]
标题的价值，不在于分得多细，而在于让结构更清楚。
:::

### 4.4 提示块

提示块本身就是我使用的段落高亮方式，常用于放置原则、提醒或结论摘要：

:::tip[写作原则]
先结论，后展开；先重点，后细节。
:::

### 4.5 表格

适合做对比、归纳、分类，例如：

| 场景 | 是否推荐 | 说明 |
| --- | --- | --- |
| 对比规则 | 推荐 | 一眼能看出差异 |
| 归纳原则 | 推荐 | 结构更清晰 |
| 叙述流程 | 视情况 | 过多时可能影响阅读 |
| 表达单一观点 | 不推荐 | 容易显得冗余 |

## 五、写作检查

写完后优先检查：

- 标题是否概括内容
- 结构是否符合总分逻辑
- 重点是否前置
- 是否存在过度拆分标题
- 是否使用了合适的 MDX 强调
- 是否让读者能快速看出主线

> <mark>目标不是把文档写得复杂，而是把复杂内容写得清楚。</mark>
