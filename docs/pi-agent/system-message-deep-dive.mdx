# 一、System Message 总览

当前版本中，最终发送给模型的 `system` message 由以下部分构成：

```
[1] Base System
    - CLI --system-prompt {arg}
    - {repo}/.pi/SYSTEM.md
    - ~/.pi/agent/SYSTEM.md
    - DEFAULT_SYSTEM_PROMPT

[2] APPEND_SYSTEM.md

[3] AGENTS.md / CLAUDE.md, concat
    - global
    - 父目录（从 root 到 cwd）

[4] CLI --append-system-prompt

[5] Runtime 注入（Skills）
```

## ✅ 最终优先级（从高到低）

| 优先级 | 来源 |
|--------|------|
| 最高 | CLI `--system-prompt` |
| ↑ | CLI `--append-system-prompt` |
| ↑ | AGENTS.md（越靠近 cwd 越高） |
| ↑ | APPEND_SYSTEM.md |
| ↑ | SYSTEM.md（project > global） |
| 最低 | DEFAULT_SYSTEM_PROMPT |

# 4.1 SYSTEM.md 发现逻辑

```ts
private discoverSystemPromptFile(): string | undefined {
  const projectPath = join(this.cwd, CONFIG_DIR_NAME, "SYSTEM.md");
  if (existsSync(projectPath)) {
    return projectPath;
  }

  const globalPath = join(this.agentDir, "SYSTEM.md");
  if (existsSync(globalPath)) {
    return globalPath;
  }

  return undefined;
}
```

✅ project 优先

## ✅ 向上遍历核心代码

```ts
function loadProjectContextFiles(...) {
  const globalContext = loadContextFileFromDir(resolvedAgentDir);

  let currentDir = resolvedCwd;
  const root = resolve("/");

  while (true) {
    const contextFile = loadContextFileFromDir(currentDir);
    if (contextFile) {
      ancestorContextFiles.unshift(contextFile);
    }

    if (currentDir === root) break;

    currentDir = resolve(currentDir, "..");
  }

  contextFiles.push(...ancestorContextFiles);
}
```

---

## ✅ 顺序保证机制

使用 `unshift()`：

- root 在前
- cwd 在后
- 越靠近 cwd 优先级越高

# 五、最佳实践指南

---

## ✅ 1️⃣ 分层组织 prompt

推荐结构：

```
~/.pi/agent/AGENTS.md        ← 全局规范
repo/AGENTS.md               ← 项目规范
repo/.pi/SYSTEM.md           ← 项目级 system 行为
```

---

## ✅ 2️⃣ 不要滥用 CLI --system-prompt

因为它会清空：

- SYSTEM.md
- APPEND_SYSTEM.md
- AGENTS.md

推荐使用：

```
--append-system-prompt
```

---

## ✅ 3️⃣ 企业环境建议

- SYSTEM.md 定义“人格与工具规则”
- AGENTS.md 定义“项目语义规则”

保持分层。
