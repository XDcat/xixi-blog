# 面向日常学习与文档整理的 Agent / Skills 分类框架

本文档将“学习资料整理、文档加工、知识管理、任务流转”拆成一组可组合的 agent / skills 类别，便于后续设计成可执行工作流。

---

## 一、总体目标

这套分类主要服务三类需求：

1. **把外部输入变成可用内容**  
2. **把已有内容变得更清晰、更统一、更可靠**  
3. **把内容整理工作流程化、自动化、可追踪**

---

## 二、分类总览表

| 类别 | 核心职责 | 典型输入 | 典型输出 | 自动化方式 | 适合场景 |
|---|---|---|---|---|---|
| 输入型 | 采集、识别、转写、抽取、初步结构化 | 网页、视频、PDF、截图、语音、聊天记录 | 原始内容结构化结果、转录文本、摘要草稿 | 半自动 / 全自动 | 把资料导入知识库 |
| 分类 / 标注型 | 打标签、归类、补元数据 | 文档、笔记、片段、任务卡 | 标签、分类、来源、优先级、难度等元数据 | 半自动 / 全自动 | 建立可检索的内容体系 |
| 总结 / 压缩型 | 提炼要点、压缩信息、分层总结 | 长文、课程、会议记录、视频转录 | 一句话摘要、要点版、详细版 | 半自动 / 全自动 | 快速理解大量资料 |
| 优化型 | 润色、改写、统一风格、补全内容 | 草稿、手写笔记、自动生成文档 | 更自然、更统一、更符合习惯的文档 | 建议 / 半自动 / 全自动 | 提高文档质量与可读性 |
| 校验 / 质量控制型 | 检查事实、格式、引用、重复、遗漏 | 待发布文档、整理结果、草稿 | 问题清单、修正建议、质检结果 | 建议 / 半自动 | 发布前审核、内容验收 |
| 检索 / 召回型 | 找回历史内容、关联相关资料 | 主题、关键词、项目名、问题描述 | 相关笔记、旧文档、引用片段 | 半自动 / 全自动 | 复用知识、找回资料 |
| 任务管理型 | 整理、发布、统计、版本与状态管理 | 文档任务、发布请求、待办清单 | 任务状态、流程结果、统计报表 | 半自动 / 全自动 | 让整理工作可追踪 |
| 协作 / 流转型 | 推动审核、分发、确认、反馈 | 待确认内容、审核请求、协作事项 | 待处理项、审批结果、反馈流转 | 半自动 | 团队协作、多人审阅 |
| 监控 / 触发型 | 监听变化、规则触发、定时触发 | 文件夹、RSS、网站更新、时间条件 | 触发事件、待处理任务 | 全自动 | 自动启动整理流程 |

## 三、逐类细化

### 1. 输入型

**职责：** 将外部信息接入系统，并做最初步的格式化和提取。

**子能力：**
- 抓取网页内容
- 视频转写
- OCR 图片识别
- PDF / 文档解析
- 语音转文本
- 聊天记录导入

**输出建议：**
- 原文
- 提取文本
- 结构化字段
- 来源信息
- 时间戳

**适用自动化程度：**
- 半自动：用户选择来源后处理
- 全自动：监听新内容并自动导入

---

### 2. 分类 / 标注型

**职责：** 给内容加上可用于管理和检索的标签与元信息。

**可标注信息：**
- 主题：编程 / 学习 / 生活 / 饮食
- 类型：笔记 / 任务 / 参考资料 / 摘要
- 来源：网页 / 视频 / PDF / 对话 / 手写
- 优先级：高 / 中 / 低
- 状态：待处理 / 已整理 / 已发布 / 已归档
- 难度：入门 / 进阶 / 深入

**价值：**
- 让内容可检索
- 让后续自动化可执行
- 让统计分析有基础

**输出建议：**
- 标签列表
- 分类结果
- 推荐目录
- 元数据 JSON

---

### 3. 总结 / 压缩型

**职责：** 从长内容中提取核心信息，减少阅读成本。

**常见形式：**
- 一句话总结
- 3~5 条要点
- 详细摘要
- 分层总结（标题层 / 要点层 / 细节层）
- 行动项提取

**与优化型的区别：**
- 总结型关注“信息保真与压缩”
- 优化型关注“表达与风格”

---

### 4. 优化型

**职责：** 对已有内容进行语言、结构、风格方面的优化。

**常见操作：**
- 润色
- 改写
- 补全
- 统一术语
- 调整语气
- 按模板重排

**输出建议：**
- 优化版正文
- 修改建议
- 风格差异说明

**自动化模式：**
- 建议模式：只给修改意见
- 半自动：生成后由用户确认
- 全自动：对低风险内容直接改写

---

### 5. 校验 / 质量控制型

**职责：** 检查内容是否正确、完整、规范、可发布。

**检查项：**
- 事实一致性
- 逻辑冲突
- 格式规范
- 标签是否缺失
- 引用是否完整
- 是否重复
- 是否遗漏关键点

**适合流程：**
- 入库前
- 发布前
- 批量处理后
- 重要文档确认前

**输出建议：**
- 问题列表
- 风险等级
- 修正建议
- 是否通过校验

---

### 6. 检索 / 召回型

**职责：** 从历史知识中找回相关资料，支持复用和关联。

**检索方式：**
- 关键词检索
- 语义检索
- 主题召回
- 时间范围召回
- 项目范围召回

**输出建议：**
- 相关文档列表
- 相关片段
- 关联理由
- 可引用内容

**价值：**
- 避免重复整理
- 复用历史经验
- 支持知识串联

---

### 7. 任务管理型

**职责：** 负责整理任务的状态推进、发布、统计与归档。

**典型任务：**
- 待整理列表管理
- 批量发布
- 归档处理
- 周报 / 月报统计
- 版本管理
- 完成状态追踪

**输出建议：**
- 任务清单
- 状态变更记录
- 统计报表
- 归档结果

---

### 8. 协作 / 流转型

**职责：** 把内容从一个处理环节推进到下一个环节，尤其适合需要人工确认的流程。

**典型动作：**
- 提交审核
- 请求确认
- 分发给指定角色
- 收集反馈
- 汇总修改意见

**适合场景：**
- 多人参与的文档整理
- 发布前审阅
- 半自动审批流

---

### 9. 监控 / 触发型

**职责：** 持续监听环境变化，并自动触发整理流程。

**常见触发条件：**
- 新文件进入目录
- 某网站内容更新
- RSS 更新
- 定时触发
- 文档数量达到阈值
- 某标签连续出现

**触发后的动作：**
- 自动导入
- 自动分类
- 自动摘要
- 自动提醒
- 自动归档

## 四、推荐的工作流分层

可以把整个系统拆成四层：

| 层级 | 包含能力 | 说明 |
|---|---|---|
| 输入层 | 输入型、监控 / 触发型 | 负责把内容接进来，或自动发现新内容 |
| 理解层 | 分类 / 标注型、总结 / 压缩型 | 负责理解内容并建立结构 |
| 处理层 | 优化型、校验 / 质量控制型、检索 / 召回型 | 负责提升质量、发现问题、召回相关知识 |
| 管理层 | 任务管理型、协作 / 流转型 | 负责推进流程、发布、统计、协作 |

---

## 五、建议的自动化级别

每一类 agent 都可以拆成三种模式：

| 模式 | 含义 | 适用场景 |
|---|---|---|
| 建议模式 | AI 只给建议，不直接改动 | 高风险内容、需要人工决策 |
| 半自动模式 | AI 先处理，用户确认后执行 | 大部分日常整理 |
| 全自动模式 | 符合规则后直接执行 | 低风险、重复性高的任务 |

## 六、推荐命名方式

建议命名时尽量体现“职责 + 场景”。例如：

- `input-web-collector`
- `input-video-transcriber`
- `tagger-topic`
- `summarizer-long-doc`
- `optimizer-style-consistent`
- `validator-pre-publish`
- `retriever-knowledge`
- `task-doc-publisher`
- `watcher-folder-trigger`

如果更偏 skills 风格，也可以用：
- `capture`
- `classify`
- `summarize`
- `refine`
- `validate`
- `retrieve`
- `manage`
- `route`
- `watch`

## 七、一个实际可落地的流程例子

以“学习一篇网页文章”为例：

1. **监控 / 触发型** 发现新网页或用户手动提交
2. **输入型** 抓取正文并清洗
3. **分类 / 标注型** 判断主题、来源、类型
4. **总结 / 压缩型** 生成摘要和要点
5. **优化型** 按你的写作习惯整理成笔记
6. **校验 / 质量控制型** 检查引用、格式、遗漏
7. **任务管理型** 标记为已处理并归档
8. **检索 / 召回型** 以后可与相关笔记关联

---

## 八、结论

目前这套分类已经比较完整，适合构建一套面向日常学习和文档整理的 agent 体系。

你可以把核心能力概括为：

- **输入**：把内容接进来
- **理解**：分类、总结、标注
- **加工**：优化、校验、召回
- **流转**：管理、协作、触发

如果下一步要继续推进，建议继续细化成：
1. 每类 agent 的输入 / 输出 schema
2. 每类 agent 的触发条件
3. 每类 agent 的权限边界
4. 一套统一的命名和目录结构
